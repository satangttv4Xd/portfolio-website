# คู่มือการใช้งานเว็บพอร์ตโฟลิโอ (ภาษาไทย)

เว็บไซต์นี้แบ่งเป็น 2 ส่วนทำงานร่วมกัน:

- **frontend/** — หน้าเว็บที่ผู้เข้าชมเห็น (React) มีปุ่มแก้ไขเนื้อหาและปุ่มดาวน์โหลด PDF
- **backend/** — เซิร์ฟเวอร์เล็ก ๆ ที่เก็บข้อมูลพอร์ตโฟลิโอและรับไฟล์รูปภาพที่อัปโหลด

ข้อมูลทั้งหมด (ชื่อ ผลงาน ทักษะ รูปภาพ ฯลฯ) ถูกเก็บเป็นไฟล์ในเครื่องของคุณเอง ไม่ได้ส่งไปที่ใด ทำให้แก้ไขและสำรองข้อมูลได้อิสระ

## 1. สิ่งที่ต้องมีก่อนเริ่ม

ติดตั้งโปรแกรมต่อไปนี้ในเครื่องก่อน (ทำครั้งเดียว):

- **Node.js** เวอร์ชัน 18 ขึ้นไป — ดาวน์โหลดได้ที่ [nodejs.org](https://nodejs.org) (เลือกเวอร์ชัน LTS)
- ตรวจสอบว่าติดตั้งสำเร็จ โดยเปิด Terminal / Command Prompt แล้วพิมพ์:
  ```
  node -v
  npm -v
  ```
  ถ้าเห็นเลขเวอร์ชันแสดงว่าใช้ได้แล้ว

ไม่จำเป็นต้องติดตั้งฐานข้อมูลใด ๆ เพิ่มเติม เพราะระบบเก็บข้อมูลเป็นไฟล์ JSON ธรรมดาให้อัตโนมัติ

## 2. ติดตั้งและเปิดเว็บไซต์ครั้งแรก

ต้องเปิด Terminal **2 หน้าต่าง** พร้อมกัน หน้าต่างหนึ่งรัน backend อีกหน้าต่างรัน frontend (ต้องรันทั้งคู่พร้อมกันเสมอเวลาแก้ไข/ดูเว็บไซต์)

### 2.1 ติดตั้งและรัน backend

```
cd portfolio-website/backend
npm install
npm run dev
```

เมื่อสำเร็จจะเห็นข้อความ `✅ Portfolio backend กำลังทำงานที่ http://localhost:5000` — ปล่อยหน้าต่างนี้ทำงานต่อไป **ห้ามปิด**

### 2.2 ติดตั้งและรัน frontend (เปิด Terminal หน้าต่างใหม่)

```
cd portfolio-website/frontend
npm install
npm run dev
```

เมื่อสำเร็จจะเห็นลิงก์ เช่น `http://localhost:5173` ให้กดลิงก์นั้นหรือคัดลอกไปเปิดในเบราว์เซอร์

> ครั้งแรกที่เปิด เว็บไซต์จะมีข้อมูลตัวอย่าง (placeholder) ให้อยู่แล้ว ให้กดปุ่ม **"แก้ไขเว็บไซต์"** ที่มุมขวาบนเพื่อเริ่มใส่ข้อมูลของคุณเองได้ทันที

### ครั้งต่อ ๆ ไป

แค่เปิด Terminal สองหน้าต่างแล้วสั่ง `npm run dev` ในแต่ละโฟลเดอร์ (ไม่ต้อง `npm install` ซ้ำอีก ยกเว้นมีการเพิ่มแพ็กเกจใหม่)

## 3. วิธีใช้งานปุ่ม "แก้ไขเว็บไซต์"

1. กดปุ่ม **แก้ไขเว็บไซต์** (ไอคอนดินสอ มุมขวาบน) จะมีแผงเลื่อนออกมาจากด้านขวา
2. ด้านบนแผงมีแท็บ 6 แท็บ:
   - **โปรไฟล์** — ชื่อ ตำแหน่ง รูปโปรไฟล์ อีเมล เบอร์โทร ที่อยู่ ลิงก์โซเชียล
   - **เกี่ยวกับ** — ข้อความแนะนำตัว และตัวเลขไฮไลต์ (เช่น ปีประสบการณ์)
   - **ทักษะ** — รายการทักษะพร้อมแถบระดับความชำนาญ
   - **ผลงาน** — การ์ดผลงานแต่ละชิ้น พร้อมรูปปก คำอธิบาย ลิงก์ และแท็ก
   - **ประสบการณ์** — เส้นทางการทำงาน เรียงตามช่วงเวลา
   - **ติดต่อ** — หัวข้อและข้อความท้ายเว็บไซต์
3. แก้ไขข้อมูลในแท็บที่ต้องการ การพิมพ์จะยังไม่บันทึกทันที (เป็นฉบับร่าง) จนกว่าจะกด **"บันทึกการเปลี่ยนแปลง"** ด้านล่างแผง
4. กด **บันทึกการเปลี่ยนแปลง** เมื่อพอใจแล้ว ระบบจะส่งข้อมูลไปเก็บที่ backend และอัปเดตหน้าเว็บทันที
5. ปุ่มรูปลูกศรหมุน (มุมล่างขวาของแผง) ใช้ **คืนค่าข้อมูลทั้งหมดกลับเป็นค่าเริ่มต้น** — ระบบจะถามยืนยันก่อนเสมอ เพราะกู้คืนไม่ได้

### วิธีอัปโหลดรูปภาพ

ในแท็บ **โปรไฟล์** หรือ **ผลงาน** จะมีกรอบรูปประแบบเส้นประ ให้ **คลิก** ที่กรอบเพื่อเลือกไฟล์ หรือ **ลากไฟล์รูป** มาวางบนกรอบได้เลย รองรับไฟล์ JPG, PNG, WEBP, GIF, SVG ขนาดไม่เกิน 8MB ต่อไฟล์ เมื่ออัปโหลดสำเร็จรูปจะอัปเดตให้ทันทีในกรอบ (ต้องกด "บันทึกการเปลี่ยนแปลง" อีกครั้งเพื่อให้ข้อมูลส่วนอื่นที่แก้ไขพร้อมกันถูกบันทึกด้วย)

### วิธีเพิ่ม/ลบรายการ (ทักษะ, ผลงาน, ประสบการณ์, ตัวเลขไฮไลต์)

แต่ละแท็บที่เป็นรายการจะมีปุ่ม **"+ เพิ่ม..."** ด้านบนสำหรับเพิ่มรายการใหม่ และปุ่ม **"ลบรายการนี้"** สีแดงใต้แต่ละรายการสำหรับลบ ในแท็บผลงานมีปุ่มลูกศรขึ้น-ลงไว้สลับลำดับการแสดงผลของผลงานแต่ละชิ้นด้วย

## 4. วิธีดาวน์โหลดพอร์ตโฟลิโอเป็น PDF

กดปุ่ม **PDF** ที่แถบเมนูด้านบน (ไอคอนลูกศรลง) ระบบจะแปลงหน้าเว็บทั้งหมด (ตั้งแต่ Hero จนถึง Contact) เป็นไฟล์ PDF หลายหน้าตามขนาดกระดาษ A4 แล้วดาวน์โหลดให้อัตโนมัติ โดยตั้งชื่อไฟล์ตามชื่อโปรไฟล์ของคุณ

เคล็ดลับ: ควรปิดแผงแก้ไขก่อนกดดาวน์โหลด เพื่อให้ PDF จับภาพเฉพาะเนื้อหาหน้าเว็บ ไม่ติดแผงแก้ไขมาด้วย

## 5. โครงสร้างไฟล์ทั้งหมด คืออะไรบ้าง

```
portfolio-website/
├── backend/
│   ├── server.js              ตัวเซิร์ฟเวอร์หลัก (API + อัปโหลดรูป)
│   ├── package.json           รายชื่อแพ็กเกจที่ backend ใช้
│   ├── data/
│   │   ├── portfolio.default.json   ข้อมูลตั้งต้น (อย่าลบ ใช้ตอนกด "คืนค่าเริ่มต้น")
│   │   └── portfolio.json           ข้อมูลจริงที่ใช้งานอยู่ (สร้างอัตโนมัติ)
│   └── uploads/                รูปภาพทุกไฟล์ที่อัปโหลดจะถูกเก็บที่นี่
│
└── frontend/
    ├── index.html              หน้า HTML หลัก โหลดฟอนต์ Google Fonts
    ├── tailwind.config.js      โทนสี/ฟอนต์ทั้งเว็บไซต์ (แก้สีธีมที่นี่)
    ├── src/
    │   ├── App.jsx             ไฟล์หลักที่ประกอบทุกหมวดเข้าด้วยกัน
    │   ├── index.css           สไตล์พื้นฐาน + อนิเมชัน scroll-reveal
    │   ├── api/api.js          ฟังก์ชันเรียก backend (ดึง/บันทึก/อัปโหลด)
    │   ├── utils/exportPdf.js  ฟังก์ชันแปลงหน้าเว็บเป็น PDF
    │   ├── hooks/useScrollReveal.js   อนิเมชันค่อย ๆ ปรากฏเมื่อเลื่อนจอ
    │   ├── components/
    │   │   ├── Navbar.jsx, Footer.jsx
    │   │   ├── EditModePanel.jsx      แผงแก้ไขเว็บไซต์ (ตัวควบคุมแท็บ)
    │   │   ├── EditPanel/              ฟอร์มแก้ไขของแต่ละแท็บ (1 ไฟล์ต่อ 1 แท็บ)
    │   │   ├── sections/               แต่ละหมวดที่แสดงบนหน้าเว็บจริง
    │   │   │   (Hero, About, Projects, Skills, Experience, Contact)
    │   │   └── ui/                     ชิ้นส่วน UI ตกแต่งสไตล์ React Bits
    │   │       (SpotlightCard, ParticleField, ShinyButton, SectionLabel ฯลฯ)
```

**กฎจำง่าย:** อยากแก้ "หน้าตา" ของหมวดไหน ให้ไปที่ `components/sections/` อยากแก้ "ฟอร์มแก้ไข" ของหมวดไหน ให้ไปที่ `components/EditPanel/` อยากแก้ "สีหรือเอฟเฟกต์ตกแต่ง" ให้ไปที่ `components/ui/` หรือ `tailwind.config.js`

## 6. แก้ไขสีธีม ฟอนต์ และดีไซน์

### เปลี่ยนชุดสี

เปิดไฟล์ `frontend/tailwind.config.js` มองหาส่วน `colors` จะเห็นกลุ่มสีตั้งชื่อไว้ เช่น

```js
amber: {
  DEFAULT: "#D9772E", // สีเน้นหลักของเว็บไซต์ (ปุ่ม, ตัวเลข, เส้นไฮไลต์)
  soft: "#6B4226",
  glow: "#F2A65A",
},
archive: {
  bg: "#0E0D0C",       // สีพื้นหลังเว็บไซต์ทั้งหมด
  panelSolid: "#1A1816", // สีพื้นหลังการ์ด/แผงแก้ไข
  line: "#2A2722",     // สีเส้นคั่น
},
```

แก้รหัสสี (hex code) เหล่านี้แล้วบันทึก หน้าเว็บจะเปลี่ยนสีทันทีโดยไม่ต้องแก้ไฟล์อื่นเลย แนะนำให้เปลี่ยนทีละสีแล้วดูผลก่อนเปลี่ยนสีถัดไป

### เปลี่ยนฟอนต์

ฟอนต์หลักโหลดมาจาก Google Fonts ในไฟล์ `frontend/index.html` (บรรทัดที่มี `fonts.googleapis.com`) หากต้องการเปลี่ยนฟอนต์:

1. ไปที่ [fonts.google.com](https://fonts.google.com) เลือกฟอนต์ใหม่ แล้วคัดลอกลิงก์ `<link>` ที่เว็บไซต์ให้มา มาแทนที่ลิงก์เดิมใน `index.html`
2. เปิดไฟล์ `frontend/tailwind.config.js` ส่วน `fontFamily` แล้วใส่ชื่อฟอนต์ใหม่แทนของเดิม เช่น

```js
fontFamily: {
  display: ["'ชื่อฟอนต์ใหม่'", "serif"], // ใช้กับหัวข้อใหญ่
  body: ["'Inter'", "sans-serif"],        // ใช้กับเนื้อหาทั่วไป
  mono: ["'IBM Plex Mono'", "monospace"], // ใช้กับป้ายเลข/แท็ก
},
```

## 7. แก้ไของค์ประกอบสไตล์ React Bits (อนิเมชันต่าง ๆ)

เว็บไซต์นี้ใช้ชิ้นส่วนตกแต่งที่เขียนขึ้นเองในสไตล์ React Bits (เว็บคอมโพเนนต์สำเร็จรูปที่ [reactbits.dev](https://reactbits.dev)) อยู่ในโฟลเดอร์ `frontend/src/components/ui/` ได้แก่:

| ไฟล์ | หน้าที่ |
|---|---|
| `ParticleField.jsx` | ฝุ่นอนุภาคลอยพื้นหลัง Hero — แก้จำนวนได้ที่ props `count` |
| `SpotlightCard.jsx` | การ์ดที่มีแสงไล่ตามเมาส์ ใช้กับการ์ดผลงาน |
| `ShinyButton.jsx` | ปุ่มที่มีแสงวิ่งผ่าน ใช้กับปุ่ม "ปิดแก้ไข" และ "ส่งอีเมลถึงฉัน" |
| `SectionLabel.jsx` | ป้ายเลขลำดับหมวด เช่น "01 — เกี่ยวกับ" |

หากต้องการนำคอมโพเนนต์จาก [reactbits.dev](https://reactbits.dev) ตัวจริงมาใช้แทน ให้เลือกคอมโพเนนต์ที่ต้องการในเว็บไซต์นั้น คัดลอกโค้ดมาวางเป็นไฟล์ใหม่ในโฟลเดอร์ `components/ui/` แล้วนำไป `import` ใช้แทนไฟล์เดิมในหมวดที่ต้องการ (เช่นใน `Hero.jsx` หรือ `Projects.jsx`) — บางคอมโพเนนต์อาจต้องติดตั้งไลบรารีเพิ่ม เช่น `framer-motion` ด้วยคำสั่ง `npm install framer-motion` ในโฟลเดอร์ `frontend`

### ปรับความถี่/ความแรงของอนิเมชัน

- อนุภาคพื้นหลัง: แก้ตัวเลขใน `<ParticleField count={70} />` ที่ไฟล์ `Hero.jsx` (ตัวเลขมาก = อนุภาคเยอะขึ้น)
- ความเร็วแสงวิ่งของปุ่ม: แก้ที่ `tailwind.config.js` ส่วน `animation.shimmer` (ตัวเลขวินาทีน้อยลง = วิ่งเร็วขึ้น)
- อนิเมชันค่อย ๆ ปรากฏเมื่อเลื่อนจอ: ควบคุมด้วยคลาส `.reveal` ใน `index.css`

## 8. เพิ่ม/ลบ/ย้ายหมวดเนื้อหา

### เพิ่มหมวดใหม่ เช่น "รางวัลที่ได้รับ"

1. สร้างไฟล์ใหม่ เช่น `frontend/src/components/sections/Awards.jsx` โดยทำตามโครงสร้างของไฟล์ `Experience.jsx` ที่มีอยู่แล้ว (คัดลอกมาแล้วแก้ข้อความ)
2. เปิด `frontend/src/App.jsx` แล้ว `import Awards from "./components/sections/Awards.jsx";` ด้านบน และวาง `<Awards ... />` ในตำแหน่งที่ต้องการระหว่างหมวดอื่น ๆ
3. ถ้าต้องการให้แก้ไขเนื้อหาหมวดนี้ผ่านปุ่ม "แก้ไขเว็บไซต์" ได้ด้วย ให้เพิ่ม field ใหม่ในไฟล์ `backend/data/portfolio.default.json`, สร้างฟอร์มใหม่ในโฟลเดอร์ `EditPanel/` และเพิ่มแท็บใหม่ในไฟล์ `EditModePanel.jsx` (ดูตัวอย่างจากแท็บ "ประสบการณ์")

### ลบหมวดที่ไม่ต้องการ

ลบ (หรือคอมเมนต์ปิด) บรรทัดที่เรียกใช้คอมโพเนนต์นั้นใน `App.jsx` เช่นลบ `<Experience ... />` ออก หมวดนั้นก็จะหายไปจากหน้าเว็บทันที โดยไม่กระทบหมวดอื่น

### ย้ายลำดับหมวด

สลับตำแหน่งบรรทัด `<Hero />`, `<About />`, `<Projects />` ฯลฯ ใน `App.jsx` ตามลำดับที่ต้องการให้แสดงผล — อย่าลืมแก้เลขลำดับ (`index="01"`, `index="02"` ...) ในแต่ละไฟล์ `sections/*.jsx` ให้เรียงใหม่ตามลำดับจริงด้วย

## 9. สำรองและกู้คืนข้อมูล

ข้อมูลทั้งหมดอยู่ใน 2 ตำแหน่งนี้ในโฟลเดอร์ `backend/`:

- `data/portfolio.json` — ข้อความ/ข้อมูลทั้งหมดที่กรอกผ่านปุ่มแก้ไข
- `uploads/` — ไฟล์รูปภาพทุกรูปที่เคยอัปโหลด

**วิธีสำรองข้อมูล:** คัดลอกทั้งสองรายการนี้ไปเก็บไว้ที่อื่น (เช่น Google Drive) เป็นระยะ

**วิธีกู้คืนข้อมูล:** นำไฟล์/โฟลเดอร์ที่สำรองไว้กลับมาวางทับตำแหน่งเดิมในโฟลเดอร์ `backend/` แล้วรีสตาร์ตเซิร์ฟเวอร์ backend

**วิธีย้ายเว็บไซต์ไปเครื่องอื่น:** คัดลอกโฟลเดอร์ `portfolio-website` ทั้งหมดไปเครื่องใหม่ แล้วทำตามขั้นตอนข้อ 2 (ติดตั้งและรัน) ใหม่อีกครั้ง ข้อมูลเดิมจะติดไปด้วยเพราะอยู่ในไฟล์ `data/portfolio.json` และโฟลเดอร์ `uploads/` อยู่แล้ว

## 10. นำเว็บไซต์ขึ้นออนไลน์ (Deploy)

เมื่อพร้อมเผยแพร่เว็บไซต์ให้คนอื่นเข้าชมได้จริง แนะนำแยก deploy 2 ส่วน:

### Frontend
1. ในโฟลเดอร์ `frontend` รันคำสั่ง `npm run build` จะได้โฟลเดอร์ `dist/` ที่พร้อมใช้งาน
2. นำโฟลเดอร์ `dist/` ไปอัปโหลดขึ้นบริการโฮสต์เว็บสถิต เช่น Vercel, Netlify หรือ Cloudflare Pages

### Backend
1. นำโฟลเดอร์ `backend` ทั้งหมดไป deploy บนบริการที่รัน Node.js ได้ เช่น Render, Railway หรือ VPS ของคุณเอง
2. ตั้งค่า environment variable `PORT` ตามที่บริการนั้นกำหนด (ถ้าจำเป็น)

### เชื่อมสองฝั่งเข้าด้วยกัน
แก้ไฟล์ `frontend/.env.example` ให้เป็น `.env` แล้วใส่ที่อยู่จริงของ backend ที่ deploy ไว้ เช่น

```
VITE_API_URL=https://your-backend-domain.com
```

แล้วสั่ง `npm run build` ใหม่อีกครั้งก่อนอัปโหลด `dist/` ขึ้นโฮสต์

## 11. แก้ปัญหาที่พบบ่อย

**หน้าเว็บขึ้นข้อความ "เชื่อมต่อ backend ไม่สำเร็จ"**
→ ลืมรัน `npm run dev` ในโฟลเดอร์ `backend` หรือปิดหน้าต่าง Terminal นั้นไปแล้ว ให้เปิดใหม่และรันคำสั่งอีกครั้ง

**ขึ้น error ว่า "port ถูกใช้งานอยู่" (port already in use)**
→ มีโปรแกรมอื่นใช้พอร์ต 5000 หรือ 5173 อยู่ก่อนแล้ว ปิดโปรแกรมนั้น หรือแก้พอร์ตใน `backend/server.js` (ตัวแปร `PORT`) และ `frontend/vite.config.js` (ค่า `server.port`)

**อัปโหลดรูปแล้วรูปไม่ขึ้น**
→ ตรวจสอบว่า backend ยังรันอยู่ และไฟล์ที่อัปโหลดไม่เกิน 8MB และเป็นไฟล์ภาพ (jpg, png, webp, gif, svg) เท่านั้น

**กดบันทึกแล้วข้อมูลไม่เปลี่ยน**
→ ตรวจดูว่ามีข้อความ error สีแดงในแผงแก้ไขหรือไม่ ปกติมักเกิดจาก backend ไม่ได้รันอยู่

**ดาวน์โหลด PDF แล้วเนื้อหาเพี้ยน/ตัดไม่สวย**
→ เป็นเรื่องปกติของการแปลงหน้าเว็บยาว ๆ เป็น PDF หลายหน้า ลองลดความยาวเนื้อหาบางหมวด หรือปิดแผงแก้ไขก่อนกดดาวน์โหลดเสมอ

**ต้องการล้างข้อมูลทั้งหมดกลับไปเป็นค่าเริ่มต้น**
→ กดปุ่มลูกศรหมุน (รีเซ็ต) มุมล่างขวาของแผงแก้ไข หรือถ้าต้องการล้างแบบเต็มรูปแบบ ให้ลบไฟล์ `backend/data/portfolio.json` ทิ้ง แล้วรีสตาร์ต backend ระบบจะสร้างไฟล์ใหม่จากค่าเริ่มต้นให้อัตโนมัติ
